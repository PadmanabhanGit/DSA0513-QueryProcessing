# AI-Based Query Optimization and Performance Analysis System

This repository implements a mini final-year project that:
- Analyzes SQL queries (static heuristics)
- Predicts execution time using a Random Forest model
- Executes queries against a MySQL database (optional)
- Records query execution history and metrics
- Provides rule-based optimization recommendations

### Key Code Points
- `app.py`:
  - `POST /analyze` accepts SQL, analyzes with `QueryAnalyzer`, predicts with `Predictor`, attempts to execute on MySQL, measures execution time (ms), stores history into `query_history` table, and renders `result.html` with rows and recommendations.
  - `GET /api/stats` aggregates counts and last execution trend for the dashboard.
  - CSV download (`/download_history`) and PDF export (`/export_pdf`) convenience endpoints.

- `models/analyzer.py`: uses regex heuristics to count JOIN/WHERE/GROUP/ORDER and naive table count (improvement: use `sqlparse`).
- `models/predictor.py`: loads `query_model.pkl` (joblib) and predicts; warns if feature names mismatch and falls back to a heuristic formula when model unavailable.
- `models/recommendation.py`: rules include checks for `SELECT *`, missing WHERE, joins>2, ORDER BY, GROUP BY, long queries (>300 chars), otherwise returns "Query appears optimized."

### How to Run Locally (MySQL)
1. Create and populate the database locally:

```bash
mysql -u root -p < database/schema.sql
```

2. Copy `.env.example` to `.env` and update credentials if needed:

```bash
copy .env.example .env
```

3. Create a virtual environment, install dependencies, train the model, and run the app:

```powershell
Set-Location -Path 'F:\QP\AI_Query_Optimizer'
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
.venv\Scripts\python.exe train_model.py
.venv\Scripts\python.exe app.py
```

4. Visit: http://localhost:5000
